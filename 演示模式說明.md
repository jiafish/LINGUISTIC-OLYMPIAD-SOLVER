# 演示模式說明

## 📁 檔案說明

- **index.html** - 正式版本（需要串接真實API）
- **index02.html** - 演示版本（不需要真實API，模擬操作流程）

## 🎯 演示模式功能

### 1. **檔案上傳（模擬）**
- ✅ 可以點擊上傳區域選擇真實PDF檔案
- ✅ 也可以點擊上傳區域自動填充假檔名（`demo_problem.pdf` / `demo_answer.pdf`）
- ✅ 即使不上傳真實檔案，也可以直接點擊 SOLVE 按鈕
- ✅ 支援拖放功能（如果拖放真實檔案）

### 2. **操作流程**
1. **選擇題型**：從 Tabs 選擇問題類型（預設：Phonology）
2. **上傳檔案**（可選）：
   - 點擊上傳區域選擇PDF檔案，或
   - 直接點擊上傳區域自動填充假檔名，或
   - 不上傳任何檔案直接進行下一步
3. **點擊 SOLVE 按鈕**：
   - 自動切換到 Loading 頁面
   - Fun Facts 開始輪播（每7秒切換）
   - Loading 動畫開始
4. **等待 10 秒**：
   - 10秒後自動切換到 Result 頁面
   - 顯示假的分析資料
5. **查看結果**：
   - Problem Analysis（問題分析）
   - Step-by-Step Solution（步驟解決方案）
   - Final Answer（最終答案）
6. **返回首頁**：
   - 點擊 "NEW PROBLEM" 按鈕
   - 重置表單並返回上傳頁面

### 3. **Loading 頁面功能**
- ✅ Loading 跳動動畫
- ✅ 20組 Linguistic Fun Facts 自動輪播（每7秒切換）
- ✅ CANCEL 按鈕可以中斷並返回首頁

### 4. **Result 頁面功能**
- ✅ 顯示三個區塊的假資料
- ✅ 區塊高度自動調整（300px - 1080px）
- ✅ 內容過長時可滾動
- ✅ "DOWNLOAD PDF" 按鈕（顯示提示）
- ✅ "NEW PROBLEM" 按鈕（重置並返回首頁）

## 🚀 使用方式

### 方式一：直接在瀏覽器打開
1. 打開 `index02.html` 檔案
2. 開始操作

### 方式二：使用本地伺服器（推薦）
```bash
# 在專案目錄下啟動簡單HTTP伺服器
# Python 3
python -m http.server 8000

# 或使用 Node.js
npx http-server

# 然後在瀏覽器打開
http://localhost:8000/index02.html
```

## ⚙️ 演示模式特點

1. **不需要後端API**：所有資料都是假的演示資料
2. **自動填充**：即使不上傳檔案也可以自動填充假檔名
3. **固定延遲**：10秒後自動顯示結果
4. **完整UI**：保留所有視覺效果和動畫
5. **完整流程**：可以完整體驗整個操作流程

## 📝 假資料說明

演示模式會顯示以下假資料：
- **Problem Analysis**：關於問題類型的分析說明
- **Step-by-Step Solution**：6個步驟的解決方案
- **Final Answer**：最終答案說明

這些資料會根據選擇的題型動態更新。

## 🔄 與正式版本的差異

| 功能 | 正式版本 (index.html) | 演示版本 (index02.html) |
|------|---------------------|----------------------|
| API呼叫 | ✅ 真實API | ❌ 模擬（10秒延遲） |
| 檔案驗證 | ✅ 需要真實檔案 | ❌ 可選（自動填充假檔名） |
| 資料來源 | ✅ 後端API回應 | ❌ 前端假資料 |
| PDF下載 | ✅ 需要API | ❌ 僅提示訊息 |

## 💡 提示

- 演示版本僅用於展示操作流程和UI效果
- 如需測試真實功能，請使用 `index.html` 並串接後端API
- 演示模式的假資料會根據選擇的題型更新
